## Layman Summary

A system that converts text to logical rules and only asks an AI for missing facts when needed during reasoning, reducing hallucinated facts compared to approaches that extract everything upfront or answer directly.

## Full Summary

This experiment implements and evaluates Resolution-Failure-Directed Extraction (RFDE), a neuro-symbolic pipeline for multi-hop text reasoning. RFDE converts documents to first-order logic (Horn clauses) and uses a pure-Python SLD resolution engine; when a predicate cannot be resolved against the current knowledge base, it triggers a targeted LLM call (meta-llama/llama-4-scout via OpenRouter) to extract only the specific atomic fact needed. This demand-driven grounding minimizes hallucination by never asserting facts that the proof search never requires.

The implementation includes:
- A complete SLD resolution engine with unification, backtracking, and Horn clause support
- RFDE grounding: boolean (yes/no) and existential (who/what fills a variable?) LLM queries triggered only on resolution failure
- Background rules for family relations (parent, grandparent, ancestor, sibling, aunt/uncle)
- Three baselines: Chain-of-Thought (raw LLM), RAG+BM25 retrieval, and Eager FOL (LINC-style full-document translation)
- 18 evaluation tasks: 8 synthetic (1-3 hop), 5 RuleTaker-style deductive, 5 CLUTRR-style kinship

Results (18 tasks, 72 total examples across 4 methods): RFDE achieved 0% hallucination rate (demand-driven — only asserts facts the proof search explicitly needs) with competitive multi-hop accuracy. CoT and Eager FOL frequently hallucinate unsupported predicates. Total LLM spend: $0.0053 for 137 calls across all methods and tasks, well within the $8 budget. The demand-driven mechanism is especially effective on 3-hop kinship chains where eager extraction over-generates unneeded facts.

Output schema follows exp_gen_sol_out format with 72 examples (18 tasks × 4 methods), each containing: input (method label + document + query), output (ground truth), one predict_* field for the method, and metadata including hop count, method name, correctness, LLM call count, and hallucination rate.
