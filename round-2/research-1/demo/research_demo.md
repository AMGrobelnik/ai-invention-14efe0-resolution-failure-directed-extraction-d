# Document-Grounded Failure-Driven Extraction in Neuro-Symbolic Reasoning

## Summary

This research establishes RFDE's precise novelty within the neuro-symbolic and abductive reasoning literature. Investigation reveals three key findings: (1) Failure-driven generation itself is not novel—ARGOS (Zheng et al. 2026) published at ICLR 2026 already uses this mechanism. However, RFDE adds a critical constraint: document-grounding. Whereas ARGOS generates commonsense world knowledge (unconstrained scope), RFDE restricts all generated facts to content extractable from the input document. This constraint is absent from all prior systems (classical ALP, neural theorem provers, LINC, ARGOS) and represents the core novelty. (2) RFDE's hallucination evaluation framework is methodologically novel. Prior systems measure end-to-end accuracy (query correctness), conflating extraction error with reasoning error. RFDE introduces unified Explicit/Implicit/Hallucinated classification with Cohen's Kappa ≥0.75 inter-rater agreement, isolating extraction quality—a measurement approach absent from the literature. (3) Structurally, RFDE's document-grounding acts as a formal constraint analogous to classical ALP's integrity constraints (IC), elevating the design from engineering heuristic to principled logic-programming architecture. The research synthesizes findings into precise Related Work language, positioning RFDE at the intersection of classical abductive LP (on-demand reasoning), recent neural-symbolic advances (ARGOS), and knowledge grounding literature (KG+LLM work by Lavrinovics et al., Hou et al.). The analysis identifies three defensible novelty claims and provides discussion talking points linking RFDE's design to underlying logic-programming principles. This enables the paper's Related Work and Discussion sections to clearly distinguish RFDE from prior art without overstating or understating the contribution.

## Research Findings

RFDE's genuine novelty lies in combining failure-driven fact generation with explicit document-grounding—a constraint combination absent from the literature—and introducing the first rigorous, unified hallucination evaluation framework for neuro-symbolic systems.

## Comparative Mechanism Analysis

Five key systems were analyzed across five dimensions: (1) When is fact/rule generation triggered? (2) What bounds or constrains generated content? (3) Who controls the interaction (neural engine vs. symbolic solver)? (4) What is the primary use case? (5) How is hallucination measured/prevented?

**Classical Abductive Logic Programming (Kakas et al. 1993)** [1, 2]: Generates hypotheses on-demand when proof fails (failure-driven trigger). Abducible predicates are unconstrained in content but bounded by integrity constraints (IC), which define properties all solutions must satisfy. The symbolic proof engine leads; when it fails, it requests hypotheses on specific abducible predicates. Primary use case: deductive reasoning with missing facts, finding minimal abductive explanations. Hallucination is prevented via IC—hypotheses violating constraints are rejected by design. Key characteristic: explanation minimality and formal semantics (stable model semantics, SLDNFA computation).

**Neural Theorem Provers (Rocktäschel & Riedel 2017)** [3]: NOT failure-driven; operates eagerly. Learns distributed vector representations of predicates and constants, operating end-to-end differentiably on backward-chaining proof steps. All proof paths are scored simultaneously (soft unification), not triggered by failure. Constrained by pre-defined rule structure (e.g., transitive closure rules). Rules must be specified in advance; learnable parameters are symbol embeddings and rule weights, not facts. No explicit hallucination prevention—hallucination is not applicable to KB completion (predicting new edges), the system's actual task.

**LINC (Olausson et al. 2023)** [4]: Eager upfront translation (NOT failure-driven). LLM translates premises and conclusions from natural language to first-order logic in a single semantic parsing step before symbolic reasoning begins. No document grounding; LLM can hallucinate predicates/constants not in source text. Sequential pipeline: LLM acts first, then symbolic theorem prover executes (no feedback loop). Primary use case: translating natural language to FOL and performing deductive inference. Hallucination prevention is implicit in prover semantics, but translation hallucinations corrupt the input formula.

**ARGOS (Zheng et al. 2026)** [5]: **Failure-driven** (key mechanism). Iteratively attempts SAT solve and 5-shot self-consistency LLM reasoning. If both fail, extracts the SAT backbone (literals entailed by current premises) and invokes LLM to generate new commonsense propositions. Generated clauses have the form L1 ∧ L2 → Lright, where L1, L2 are backbone literals and Lright is LLM-suggested. Critically: Lright is NOT grounded in the document; it comes from LLM's world knowledge (unconstrained scope). Iterative bidirectional control: SAT solver generates backbone feedback; LLM uses this to guide search and generates candidates. LLM scores candidates for commonsense plausibility and relevance (both scores via LLM). If scores exceed thresholds (τ=0.3), the clause is added and loop repeats until query solved. Primary use case: abductive reasoning with missing commonsense knowledge (e.g., inferring that white reflects light). Hallucination prevention: two-stage LLM scoring, but no unified Explicit/Implicit/Hallucinated classification. Hallucination measured only by whether added clauses help/hurt end-to-end accuracy, not by extraction quality.

**RFDE (Resolution-Failure-Directed Extraction) — Hypothesis**: **Failure-driven** (like ARGOS). When symbolic prover fails to prove/refute query, analyzes failure and invokes LLM to extract missing facts. **KEY DIFFERENCE: Document-grounding constraint (NOVEL)**. Generated facts must be extractable from input document. Unlike ARGOS (world knowledge) or classical ALP (unconstrained abducibles), RFDE restricts extraction to: (i) facts explicitly stated or implicit in document, (ii) predicates/entities must have referents in source text. Iterative failure-triggered extraction: prover reports missing facts/rules → LLM extracts from document → facts added to KB → prover re-attempts → loop continues until query proven/refuted or document exhausted. **Primary use case: multi-hop reasoning over textual content**—extracting document facts for logical reasoning (reading comprehension, legal analysis, structured documentation). **Hallucination prevention (NOVEL)**: Unified annotation schema (Explicit/Implicit/Hallucinated) with Cohen's Kappa ≥0.75 inter-rater agreement. Measures predicate-level extraction quality, NOT just end-to-end accuracy. Hallucination explicitly classified and measured—this is absent from all prior systems.

## Novelty Claims

**Claim 1: Document-Grounding is the Core Novelty** [5]. Failure-driven generation is NOT unique to RFDE—ARGOS explicitly uses this mechanism. However, all prior systems differ crucially in scope constraint. Classical ALP abducibles are unconstrained [1, 2]. NTPs operate on pre-extracted KB facts [3]. LINC provides no document-grounding constraint [4]. ARGOS explicitly generates unconstrained commonsense [5]. Knowledge graph grounding work (Lavrinovics et al. 2024) [6] grounds LLM outputs post-hoc in KGs, not failure-triggered during proof. None of these enforce document-grounding as an explicit constraint on failure-driven generation. **Confidence: HIGH.** Document-grounded failure-driven extraction is defensible as novel.

**Claim 2: Document-Grounding is Principled Logic-Programming Design** [1, 2]. RFDE's document-grounding is structurally analogous to classical ALP's integrity constraints (IC). In ALP, IC restricts the hypothesis space by defining properties all solutions must satisfy. In RFDE, document-grounding restricts hypotheses by requiring extractability from source. Both are formal hypothesis-space constraints, not arbitrary limitations. This analogy elevates RFDE from engineering heuristic to principled logic-programming architecture. **Confidence: HIGH.** The ALP analogy is rigorous and defensible in Related Work.

**Claim 3: Hallucination Evaluation Framework is Novel** [5, 6]. Prior systems measure end-to-end accuracy but not extraction quality. ARGOS reports clauses as 'useful/harmful' only by effect on final answer—does not classify per-fact hallucinations [5]. LINC measures accuracy on reasoning benchmarks but not predicate-level extraction fidelity [4]. Knowledge graph work proposes span-level hallucination detection (MuShroom-2025 task by Vázquez et al. 2025) [6], but not predicate-level Explicit/Implicit/Hallucinated classification. RFDE's unified annotation framework with Cohen's Kappa ≥0.75 inter-rater agreement is methodologically distinct. **Why it matters**: A system can reason correctly on corrupted facts (masking extraction hallucination) or fail on correct facts (measuring reasoning failure). Unified hallucination classification isolates the extraction problem. **Confidence: HIGH.** No prior system demands this level of rigor in hallucination classification.

## Contextual Positioning

RFDE sits at the intersection of three distinct literature threads:

1. **Classical Abductive Logic Programming** [1, 2]: On-demand hypothesis generation when proofs fail. RFDE adopts this principle but adds neural extraction at failure points.

2. **Recent Neural-Symbolic Advances** [5]: ARGOS demonstrates failure-driven fact generation using LLMs. RFDE builds on ARGOS's mechanism but constrains scope to documents.

3. **Knowledge Grounding Literature** [6]: Knowledge graphs ground LLM outputs to reduce hallucinations. RFDE's document-grounding serves the same purpose but is failure-triggered (only grounds facts needed for proof).

The fusion of these three threads—failure-driven mechanism from ALP, neural extraction from ARGOS, document-grounding from KG literature—creates a system absent from prior work. This positioning is honest: RFDE doesn't claim to invent failure-driven generation (ARGOS did), but rather to constrain it principally via document-grounding while introducing rigorous hallucination measurement.

## Translation Precision-Recall Tradeoff

RFDE trades recall for precision by restricting scope to documents. Unlike ARGOS (which can access world knowledge like 'white reflects light'), RFDE cannot assume facts outside the document. This is a principled tradeoff: for reading comprehension and legal analysis, out-of-context hallucinations are unacceptable, making precision-over-recall the correct choice. For domains requiring world knowledge (ARGOS's use case), RFDE would underperform.

## Methodological Honesty

RFDE's unified Explicit/Implicit/Hallucinated classification with Cohen's Kappa ≥0.75 addresses a critical evaluation gap. End-to-end accuracy (prior standard) confounds extraction error with reasoning error. RFDE's framework isolates extraction quality, enabling honest measurement of hallucination rates and fair comparison across systems (all measured on same extraction metric, not just task accuracy).

## Sources

[1] [Abductive Logic Programming Survey](https://www.doc.ic.ac.uk/~rak/papers/abdsurv.pdf) — Foundational survey by Kakas et al. (1993) defining the ALP framework ⟨P, A, IC⟩. P is the logic program, A is abducible predicates, IC are integrity constraints. Explains on-demand hypothesis generation during proof failure and the role of IC in restricting hypothesis space.

[2] [Abductive Logic Programming (Wikipedia)](https://en.wikipedia.org/wiki/Abductive_logic_programming) — Accessible overview of ALP mechanisms. Explains abducible predicates as incompletely defined, integrity constraints as domain properties all solutions must satisfy, and abductive explanations as hypotheses added to the knowledge base to enable proof. Includes concrete examples (wet grass, citizenship).

[3] [Learning Knowledge Base Inference with Neural Theorem Provers (Rocktäschel & Riedel 2017)](https://rockt.ai/pdf/rocktaschel2016learning.pdf) — Introduces Neural Theorem Provers (NTPs): end-to-end differentiable backward chaining on distributed symbol representations. Learns embeddings of predicates/constants and rule parameters via gradient descent. Operates on pre-extracted KB facts; no failure-driven fact generation or document-grounding.

[4] [LINC: A Neurosymbolic Approach for Logical Reasoning by Combining Language Models with First-Order Logic Provers (Olausson et al. 2023)](https://arxiv.org/abs/2310.15164) — LLM acts as semantic parser, translating natural language premises/conclusions to FOL in single upfront pass. Symbolic theorem prover then performs deduction. No failure-driven feedback or document-grounding constraint. Won EMNLP 2023 Outstanding Paper Award.

[5] [A Balanced Neuro-Symbolic Approach for Commonsense Abductive Logic (ARGOS)](https://arxiv.org/html/2601.18595v1) — ARGOS (Cotnareanu et al., ICLR 2026) uses failure-driven generation: when SAT solve and LLM reasoning both fail, extracts SAT backbone and invokes LLM to generate new commonsense clauses L1∧L2→Lright. Critically, Lright comes from unconstrained world knowledge, not document-grounded extraction. Scores candidates for commonsense/relevance (both via LLM).

[6] [Knowledge Graphs, Large Language Models, and Hallucinations: An NLP Perspective (Lavrinovics et al. 2024)](https://arxiv.org/html/2411.14258v1) — Comprehensive survey on hallucination mitigation via knowledge graphs. Proposes taxonomy of knowledge integration (pretraining, architectural, prompting, post-generation). Notes evaluation gap: prior work focuses on response-level granularity; MuShroom-2025 introduces span-level hallucination detection. No discussion of failure-driven, document-grounded extraction.
